# A/B Testing

A/B testing turns product decisions into experiments. Instead of guessing, you compare two or more versions under similar conditions and let the data show which one performs better.

## Why It Matters

The value of A/B testing is not just finding the right answer. It is also about ruling out weak ideas before they become expensive mistakes.

## The Seven Steps

| Step | What to Do |
|---|---|
| 1. Hypothesis | State what you expect to happen |
| 2. Metrics | Define the primary, secondary, and guardrail metrics |
| 3. Experiment Design | Decide what changes and how users are split |
| 4. Implementation | Launch the test correctly and monitor it |
| 5. Data Collection | Make sure the data is complete and clean |
| 6. Analysis | Check significance and effect size |
| 7. Decision | Accept, reject, or refine the idea |

## When to Use It

- UI and UX optimization
- Pricing experiments
- Marketing copy and landing pages
- Feature launches and product changes

## When Not to Use It

- Sample size is too small
- The test would take too long to matter
- Metrics are unclear
- The system cannot support clean user grouping

## Two Famous Examples

### Google Blue Button

Google tested many shades of blue for its links and buttons. Even a tiny difference in click-through rate mattered at Google scale.

### Amazon Guest Checkout

Amazon reduced friction by allowing guest checkout instead of forcing registration first. That change improved conversion and revenue.

## Common Mistakes

| Mistake | Why It Fails |
|---|---|
| Stopping too early | Creates false positives |
| Testing too many things at once | Makes the result hard to trust |
| Allowing sample contamination | Blurs the experiment groups |
| Ignoring Simpson's paradox | Overall results can hide segment-level truth |
| Using too little sample | Leads to weak statistical power |
