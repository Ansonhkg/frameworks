# Platform Ecosystem Canvas

The Platform Ecosystem Canvas helps teams design platform products such as social networks, marketplaces, learning platforms, and communities.

## Why It Exists

Pipeline products and platform products are not the same.

Pipeline thinking assumes a company builds something and hands it to a user. Platform thinking is about connecting producers and consumers so they create value together.

## Core Components

| Component | Meaning |
|---|---|
| Producer | Creates value |
| Value Unit | The object being exchanged |
| Filter | Matches and screens supply and demand |
| Consumer | Receives and uses the value |

Platform infrastructure also includes trust, payments, identity, and APIs.

## When to Use It

Use it when you are designing:

- Two-sided markets
- Social products
- Learning platforms
- Marketplaces
- Community platforms

## WeChat Case Study

WeChat evolved from messaging into a much larger platform ecosystem.

It started with chat, then added social sharing, then payments, then mini programs. Each addition strengthened the network and increased the platform's value.

## How to Design a Platform

1. Define the core interaction.
2. Identify all participants.
3. Define the value unit.
4. Design filters and matching.
5. Build the platform infrastructure.

## Two-Sided Market Theory

Platform thinking is supported by broader platform economics:

- Two-sided markets
- Network effects
- Ecosystem coordination
- Value chain design
