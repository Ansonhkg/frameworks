# HEART Framework

HEART is Google's user experience metrics framework. It breaks UX into five measurable dimensions: Happiness, Engagement, Adoption, Retention, and Task Success.

## Why It Matters

People often say a product "feels good" or "feels bad," but that is hard to act on. HEART turns vague UX discussion into a metric system that teams can actually use.

## The Five Dimensions

| Letter | Meaning | What It Measures |
|---|---|---|
| H | Happiness | Satisfaction, delight, and perception |
| E | Engagement | Frequency and depth of interaction |
| A | Adoption | How many users begin using the product or feature |
| R | Retention | How many users keep coming back |
| T | Task Success | Whether users can complete key tasks successfully |

## Goals, Signals, Metrics

HEART becomes practical when each dimension is translated into three steps:

| Step | Question |
|---|---|
| Goals | What does success look like? |
| Signals | What behavior or feedback suggests progress? |
| Metrics | What do we measure? |

## How to Use HEART in Product Iteration

1. Decide what the current release should improve.
2. Choose the most relevant HEART dimension.
3. Define measurable signals and metrics.
4. Set a baseline and a target.
5. Track the metric after launch.
6. Feed the result into the next iteration.

## Google Docs Example

Google used HEART to evaluate Google Docs:

- Happiness: survey satisfaction and preference
- Engagement: editing frequency and interaction depth
- Adoption: new user and new feature uptake
- Retention: repeat use over time
- Task Success: time to first successful edit and completion rate

One useful lesson from the case is that adoption and retention are related but not the same thing.

## Common Mistakes

| Mistake | Why It Fails |
|---|---|
| Treating the five dimensions as independent checkboxes | HEART works as a system |
| Ignoring Happiness | Behavioral metrics alone miss the emotional layer |
| Overcomplicating the setup too early | Start with the most important flow first |
| Using one metric to explain everything | Different dimensions need different signals |
