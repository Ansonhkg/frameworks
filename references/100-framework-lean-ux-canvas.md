# Lean UX Canvas

The Lean UX Canvas helps teams connect lean thinking and user experience design. It is built for fast iteration, shared understanding, and hypothesis-driven product work.

## Why It Matters

Teams often argue about solutions before they agree on the problem. The canvas forces the conversation to start with the user, the problem, and the measurable outcome.

## The Eight Boxes

| Box | What It Covers |
|---|---|
| Target Users | Who are we designing for? |
| Problem Statement | What problem are we trying to solve? |
| User Journey | In what context does the user experience the problem? |
| Solution Ideas | What might solve it? |
| Core Metrics | How will we know we are succeeding? |
| Assumptions | What do we need to verify? |
| MVP | What is the smallest testable version? |
| Success Criteria | What counts as success? |

## When to Use It

- Before a design sprint
- At the start of a new feature effort
- When a team needs alignment quickly
- When there is not enough time for heavy documentation

## How to Use It Well

1. Start with the target users.
2. Define the problem clearly.
3. Map the user journey.
4. Explore solution ideas.
5. Decide on the metrics that matter.
6. List the assumptions to test.
7. Build the MVP.
8. Agree on the success criteria.

## Spotify Case Study

Spotify used Lean UX-style team structures to move fast:

- Small cross-functional squads
- Clear ownership of a feature or workflow
- Rapid iteration based on feedback

This made continuous improvement and fast product learning possible.

## Common Mistakes

| Mistake | Why It Fails |
|---|---|
| Filling every box too early | The canvas is a thinking tool, not a form to complete mechanically |
| Skipping the problem statement | The team may solve the wrong thing |
| Treating the canvas as static | The canvas should change as learning increases |

## Lean UX vs Related Frameworks

| Framework | Best For | Strength |
|---|---|---|
| Lean UX Canvas | Fast UX planning and hypothesis work | Simple and practical |
| Business Model Canvas | Strategy and business design | Broad business view |
| Design Sprint | Fast problem solving and prototyping | Fast, focused execution |
