# 100 Product Design Frameworks

From user insight to systems thinking, this is a core toolkit that covers the full product lifecycle. Pick any framework to open its detail page and usage steps.

## Framework Catalog

### Row 1

| # | Title | Category | Description |
|---|-------|----------|-------------|
| **01** | Design Thinking | User Insight | A five-step, human-centered innovation process: understand users, define the problem, ideate, prototype, and test. It turns abstract problems into concrete solutions. |
| **02** | User Journey Map | User Insight | Visualize the full experience across touchpoints. Identify key moments, highs and lows, and service breaks to find opportunities for improvement. |
| **03** | Empathy Map | User Insight | Build a richer user profile from four dimensions: sees, hears, thinks, and does. Deeply understand real needs and pain points. |
| **04** | JTBD (Jobs to Be Done) | User Insight | People buy products to "hire" them for a specific job, not to own the product itself. Focus on the task context instead of demographics. |
| **05** | Five-Question User Interview | User Insight | A deep follow-up interview method that starts with behavior and motivation and digs into the truth behind user stories. |

### Row 2

| # | Title | Category | Description |
|---|-------|----------|-------------|
| **06** | Kano Model | User Insight | Split features into must-have, performance, and delight factors. Identify what is required, what creates surprise, and how to allocate resources wisely. |
| **07** | Service Blueprint | User Insight | Visualize the front-stage touchpoints and back-stage support systems in a service flow. Expose bottlenecks and cross-functional friction. |
| **08** | Contextual Inquiry | User Insight | Observe, interview, and record behavior in the user's real work or life context. Gather first-hand data in a natural environment. |
| **09** | KJ Method / Affinity Diagram | User Insight | Turn large amounts of qualitative data into a visual matrix and cluster related ideas to surface patterns and structured insight. |
| **10** | Diary Study | User Insight | Ask users to record activities, feelings, and behavior changes over a set period. Capture natural behavior across time. |
