# Opportunity Solution Tree

The Opportunity Solution Tree helps teams explore, compare, and select the best solution paths during product development. It shows the path from problem to opportunity to solution in a clear visual structure.

## What It Is

The framework helps a team understand:

- The real problem
- The market opportunity hidden inside it
- Possible solutions
- How those solutions can be tested and validated

## Main Parts

1. Problem layer - the challenge the user is facing
2. Opportunity layer - the underlying opportunities the problem creates
3. Solution layer - possible responses to those opportunities
4. Validation layer - how to test the ideas

## How to Use It

1. Define the problem.
   - Use user research and market research to identify the real issue.

2. Identify opportunities.
   - Look for unmet needs, friction points, and moments of frustration.

3. Explore solutions.
   - Generate multiple ways to address each opportunity.

4. Evaluate and choose.
   - Make decisions using evidence and product judgment.

## Case Study: An E-Commerce Checkout Improvement

An e-commerce platform noticed that users were dropping out during checkout and used an Opportunity Solution Tree to find a fix.

### Process

1. Problem definition: users were abandoning checkout.
2. Opportunity identification: simplify checkout and add more payment options.
3. Solution exploration: generate multiple ways to reduce friction.
4. Evaluation and selection: choose the most feasible solution.

### Result

The platform improved conversion by 15%.

## FAQ

### Which teams should use it?

Any team that needs product planning and decision support, especially startups and fast-growing companies.

### How is solution value evaluated?

Use market size, pain intensity, technical feasibility, and similar dimensions.

### How long does it take?

Usually one to four weeks, depending on complexity.
