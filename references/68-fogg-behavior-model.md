# Fogg Behavior Model

The Fogg Behavior Model says behavior happens when motivation, ability, and a prompt arrive at the same time.

## Why Users Do Not Act

A lot of product questions are really behavior questions:

- Why do users ignore a feature?
- Why do they not complete signup?
- Why does retention stay low?

Fogg gives you a practical way to diagnose the problem.

## B = MAP

Behavior = Motivation x Ability x Prompt

| Factor | Meaning |
|---|---|
| Motivation | The user's desire to do the behavior |
| Ability | How easy the behavior is to do |
| Prompt | The trigger that tells the user to act |

If one of these is missing, the behavior usually does not happen.

## When to Use It

Use the model when you want to change behavior in:

- Product design
- Conversion optimization
- UX improvement
- Marketing and notifications

## Duolingo Case Study

Duolingo used behavior design to improve streak retention.

- Motivation: streaks and progress create a sense of achievement
- Ability: lessons are simplified so users can keep going
- Prompt: notifications remind users to return

The result was better retention and stronger daily habits.

## How to Apply It

1. Define the target behavior.
2. Measure the current motivation, ability, and prompt state.
3. Find the bottleneck.
4. Design the intervention.
5. Test and iterate.
