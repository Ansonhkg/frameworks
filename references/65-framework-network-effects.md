# Network Effects

Network effects make products more valuable as more people use them. In many products, that is the difference between a niche tool and a dominant platform.

## Why Some Products Get Stronger

The more users a product has, the more useful it becomes. That is the basic idea behind network effects.

## Four Types

| Type | Meaning | Example |
|---|---|---|
| Direct | More users directly increase value | WeChat, Skype, telephone |
| Indirect | One user group increases the value of another | Taobao, App Store |
| Data | More usage creates better data and better product quality | Google Search, Netflix, Waze |
| Two-sided | Two different user groups reinforce each other | Credit cards, dating sites, delivery platforms |

## How to Build Them

1. Define the core value proposition.
2. Choose the right network effect type.
3. Solve the chicken-and-egg problem.
4. Create a growth mechanism that reinforces itself.

## Chicken-and-Egg Strategies

- Single-point breakthrough: start with one niche and dominate it
- Fake it till you make it: simulate supply or activity early on
- Subsidize one side: make one side cheap or free to attract the other side
- Built-in network effects: use existing social graphs or contacts

## Metrics

| Metric | What it shows |
|---|---|
| User connection density | Strength of direct network effects |
| Two-sided density | Marketplace matching efficiency |
| Data contribution rate | Data-network potential |
| Retention slope | Whether network effects improve retention over time |

## Case Studies

### WeChat

WeChat grew by adding messaging, then social sharing, then a mini-program ecosystem that strengthened the network effect across users, developers, and merchants.

### Airbnb

Airbnb used a supply-first strategy, professional photos, trust mechanisms, and global expansion to strengthen its two-sided marketplace.

## Common Mistakes

1. Treating a linear process like a network effect.
2. Ignoring the network structure itself.
3. Choosing weak or unmeasurable nodes.
4. Forgetting the cold-start problem.
5. Oversimplifying a complex market.
6. Ignoring outside market forces.
