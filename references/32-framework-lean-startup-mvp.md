# Minimum Viable Product

An MVP, or Minimum Viable Product, is the smallest version of a product that can test the most important assumption. Its purpose is learning, not completeness.

## Why It Matters

Building a full product before testing the idea is risky. An MVP lets you learn early, before you spend too much time, money, or morale on the wrong direction.

## The Build-Measure-Learn Loop

| Step | What Happens |
|---|---|
| Build | Create the smallest version that can test the assumption |
| Measure | Collect real user data |
| Learn | Decide whether the idea is valid or needs to change |

## Five Common MVP Types

| Type | What It Does |
|---|---|
| Single-Feature MVP | Builds only the core feature |
| Demo / Video MVP | Tests interest with a demo or pitch video |
| No-Code MVP | Uses existing tools to build fast |
| Waitlist MVP | Measures demand before the product exists |
| Landing Page MVP | Tests interest through a simple page |

## When to Use It

- Testing a market hypothesis
- Choosing between product directions
- Validating a feature before a full rollout
- Making data-driven decisions under uncertainty
- Reducing the cost of failure

## Dropbox Case Study

Dropbox used a short demo video to validate interest in cloud storage before the full product was built.

| Stage | Action | Result |
|---|---|---|
| Build | Create a short product demo video | Very low cost |
| Measure | Share it publicly | Waitlist grew quickly |
| Learn | Confirmed demand for the idea | Helped unlock investment |

The lesson is simple: the cheapest way to learn is usually the best MVP.

## Principles for Good MVPs

1. Define the assumption you want to test.
2. Choose the MVP type that matches the question.
3. Set clear success metrics.
4. Iterate quickly instead of polishing endlessly.
5. Accept failure early if the idea is weak.
