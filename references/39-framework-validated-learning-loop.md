# Validated Learning Loop

The validated learning loop is the Build-Measure-Learn cycle at the heart of lean startup thinking. Instead of guessing, teams build something small, measure real behavior, learn from the results, and adjust direction.

## Introduction

Dropbox is a classic example. It did not start by building a full product. It started with a video that tested whether the assumption was correct.

The goal is to get the smallest possible amount of time and money spent on the biggest possible amount of learning.

## The Three Stages

| Stage | Purpose |
|---|---|
| Build | Create the smallest useful prototype or MVP |
| Measure | Collect real user data and feedback |
| Learn | Decide whether to keep going, pivot, or adjust |

## When to Use It

Use the loop whenever you need to make decisions under uncertainty:

- Startups validating market assumptions
- Product teams iterating on a feature
- Data-driven teams making decisions
- Organizations adapting to change

## Dropbox Case Study

In 2007, Drew Houston used a short video to validate the market for cloud storage.

| Metric | Value |
|---|---|
| Waitlist growth | From 50 people to 75,000 |
| Conversion | About 60% of viewers joined the waitlist |
| Build cost | Essentially zero for the prototype |

The video proved that users wanted seamless file syncing before the product existed in full.

## How to Run One Loop

1. Build the hypothesis.
   - Write down what you believe.

2. Design the MVP.
   - Use the simplest possible way to test the belief.

3. Measure the data.
   - Track the numbers that matter.

4. Learn the insight.
   - Understand why users behaved that way.

5. Adjust direction.
   - Use what you learned to improve the product strategy.

| 2021 | The loop is standard practice in many startups |
