# Impact/Effort Matrix

The Impact/Effort Matrix helps teams rank tasks by comparing the value of the outcome against the work required to get it done. It is a quick way to find high-value, low-effort opportunities.

## Why It Matters

Without a shared scoring model, prioritization becomes a shouting match.

The matrix gives everyone the same two questions:

- How much impact will this have?
- How much effort will it take?

## The Four Quadrants

| Quadrant | Name | Meaning | Action |
|---|---|---|---|
| High impact, low effort | Quick Wins | Best return for the least work | Do immediately |
| High impact, high effort | Major Projects | Valuable but expensive | Plan carefully |
| Low impact, low effort | Fill-ins | Small tasks that can fill gaps | Do when time allows |
| Low impact, high effort | Time Sinks | Poor return on investment | Do not do |

## When to Use It

This matrix is useful for:

- Quarterly planning
- Bug fix prioritization
- Growth experiment selection
- Team decision meetings
- Time management
- New project scoring

## Basecamp Example

Basecamp used this kind of matrix to simplify roadmap planning and decide which ideas should be moved forward.

The result was a more honest view of the work:

- Quick wins were shipped first
- Major projects were scheduled deliberately
- Low-value work was dropped
- Small filler tasks were kept only when they were worth the time

## Five Reminders

1. Define what "impact" means before the meeting.
2. Start with smaller decisions and build momentum.
3. Re-score the matrix when priorities change.
4. Do not choose work just because it is easy.
5. Make sure every quadrant is used with purpose.
