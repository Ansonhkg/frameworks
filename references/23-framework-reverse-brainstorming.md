# Reverse Brainstorming

Reverse brainstorming is the habit of solving a problem by first asking how to make it worse. By inverting the question, teams often discover risks, assumptions, and hidden solution paths they would otherwise miss.

## Introduction

When teams get stuck in standard brainstorming, they tend to produce the same kind of answers. Reverse thinking breaks that pattern.

Edward de Bono described this kind of sideways thinking as a way to move around a problem when a direct approach is not working. The value is simple: by exploring failure first, you often find the path to success.

## The Three-Step Method

1. Define the worst case
   - Ask: if we wanted this project to fail completely, what would we do?

2. List the failure factors
   - Technical
   - Team-related
   - Market-related
   - Resource-related

3. Reverse each factor into a solution
   - Ask: what would we do to avoid that failure?

## When to Use It

| Scenario | Fit |
|---|---|
| Product innovation is stuck | Very high |
| The team is locked in a fixed mindset | Very high |
| You need risk prevention | High |
| You are planning strategy | High |

Best used when:

- Standard brainstorming keeps producing the same ideas
- You need a breakthrough solution
- You are identifying risks before a project starts

## Amazon Case Study

Amazon often used a pre-mortem mindset before launching major initiatives: assume the project has already failed, then work backward from the causes.

For AWS, the team could have asked what would make the service fail:

- It would be too expensive for smaller companies
- It would be unstable and constantly down
- The documentation would be too hard to understand

The reversed solutions were straightforward:

- Pay-as-you-go pricing
- Strong uptime guarantees
- Detailed documentation and examples

## Five Keys

1. Create psychological safety so people can share ugly ideas.
2. Put a strict time limit on each phase.
3. Invite cross-functional participants.
4. Use visual tools like sticky notes and whiteboards.
5. Capture every idea without judgment.

## Innovation Timeline

| Year | Example | Reverse Question | Result |
|---|---|---|---|
| 1984 | Macintosh | How can a computer be hardest to use? | Graphical interfaces made it easier |
| 2007 | iPhone | How can a phone be least convenient? | Touch interaction changed mobile design |
| 2015 | Tesla Model 3 | How can an EV feel least attractive? | Software and autonomy became a differentiator |
| 2020 | Zoom | How can video meetings be hardest to use? | One-click joining became a core advantage |
